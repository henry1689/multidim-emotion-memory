# 数字大脑任务拆解 v0.1

## 1. 任务编号规则

- `DB-Axx`：架构与原则
- `DB-Mxx`：对象模型与记忆模型
- `DB-Kxx`：知识块与脑补
- `DB-Gxx`：图谱与链路
- `DB-Rxx`：召回与重建
- `DB-Uxx`：前端 demo
- `DB-Txx`：测试与验收
- `DB-Oxx`：运营、台账、文档

## 2. 任务清单

### DB-A01 总体原则冻结

- 目标：冻结项目顶层定义、边界、双脑补架构和三层记忆模型
- 要求：
  - 明确系统是什么，不是什么
  - 明确事实层、叙事层、推断层
  - 明确在线脑补与离线脑补职责
- 产出：
  - 总架构文档
  - 边界原则文档
- 验收：
  - 团队可用一页文档讲清系统边界
  - 后续任务不再反复改项目定义

### DB-M01 对象模型冻结

- 目标：冻结第一版核心对象与关系
- 要求：
  - 确定 `Session`、`Utterance`、`Cue`、`Signal`、`Interpretation`、`MemoryEpisode` 等对象
  - 区分形成链和关联链
- 产出：
  - 对象字典
  - 关系字典
- 验收：
  - 任一示例文本可映射到对象模型

### DB-M02 九维编码规范

- 目标：明确九维字段、强度、置信、引用规则
- 要求：
  - 每维定义字段、分值、引用格式
  - 支持来源与不确定性标注
- 产出：
  - 九维字段字典
- 验收：
  - 样例 episode 可稳定输出九维结果

### DB-K01 字典式记忆块

- 目标：建立 `DictionaryBlock` 与 `EntityCard` 机制
- 要求：
  - 支持人物、作品、术语、事件等公共对象
  - 支持简介块复用
- 产出：
  - 字典块对象结构
  - 优先级与缓存策略
- 验收：
  - 同一对象二次提及时不再重复生成长解释

### DB-K02 在线脑补

- 目标：在当前输入阶段即时激活 Primer
- 要求：
  - 支持公共对象识别
  - 输出极简背景备注
  - 备注参与当前 episode 编码
- 产出：
  - 在线脑补流程
  - Primer 数据结构
- 验收：
  - 提及公共对象时，前端可即时显示备注卡

### DB-K03 离线脑补

- 目标：新增会话后重整模块
- 要求：
  - 扫描会话
  - 补充字典块
  - 重算关联
  - 生成新版本
- 产出：
  - 离线脑补流程
  - 版本链规则
- 验收：
  - 不覆盖旧版本
  - 新增块和关系可回溯

### DB-G01 图谱模型

- 目标：定义节点、边、维度标签和 path 查询结构
- 要求：
  - 节点类型最少覆盖 episode、人物、事件、情感、主题、场景、时间
  - 边明确来自哪一维
- 产出：
  - 图谱模型文档
- 验收：
  - 当前记忆局部图可正确展示

### DB-G02 时序链与情绪链

- 目标：定义事件流、情绪曲线、关键帧机制
- 要求：
  - 支持前后节点关系
  - 支持情绪拐点与事件触发点
- 产出：
  - 时序链模型
  - 情绪链模型
- 验收：
  - 一段样例文本可输出事件链和情绪变化

### DB-R01 模式分离

- 目标：建立 episode 边界和 9D 指纹规则
- 要求：
  - 支持相似经历分离
  - 支持 append / new / branch 三种判断
- 产出：
  - 指纹规则
  - 边界判断规则
- 验收：
  - 两次相似咖啡厅经历不会错误合并

### DB-R02 模式补全

- 目标：建立片段线索召回到完整场景还原的主链路
- 要求：
  - 多入口查询
  - 候选排序
  - 缺失维度补全
- 产出：
  - 场景重建规范
- 验收：
  - 能从少量线索重建目标记忆

### DB-U01 Demo 单页框架

- 目标：建立单页多面板总布局
- 要求：
  - 输入区
  - 9D 可视化
  - 时序链
  - 关联图谱
  - 情绪曲线
  - 场景还原
- 产出：
  - 组件树
  - 面板职责定义
- 验收：
  - 页面信息结构稳定，不反复换布局

### DB-U02 样例数据与示例剧本

- 目标：准备最小样例集
- 要求：
  - 至少覆盖会议场景
  - 至少覆盖亲密关系场景
  - 至少覆盖公共对象脑补场景
- 产出：
  - demo fixtures
- 验收：
  - 所有面板都能基于样例跑通

### DB-U03 前端联动

- 目标：让 9D、图谱、时序、情绪、还原面板互相联动
- 要求：
  - 选中 episode 联动所有面板
  - 点击节点高亮链路和说明
- 产出：
  - 联动规则
- 验收：
  - 任一面板交互都能同步更新主视图

### DB-T01 验收基线

- 目标：建立项目一期验收标准
- 要求：
  - 定义成功样例
  - 定义失败样例
  - 定义不可接受行为
- 产出：
  - 测试清单
- 验收：
  - 团队对“算完成”达成一致

### DB-O01 项目台账

- 目标：建立任务总览、状态、问题点、时间表
- 要求：
  - 统一编号
  - 统一状态
  - 统一时间轴
- 产出：
  - 项目总览表
  - 风险与问题表
- 验收：
  - 团队可直接按台账推进

# 多维情感知识库

## 当前定位

该项目目标不是做普通笔记库，而是做一个能够从多维线索中还原事件、人物、情绪、场景与关系的记忆引擎。

## 当前升级方向

项目 framing 已从“9D 标签库”升级为“类海马体情景记忆系统”。

当前重点不再只是多维标注，而是以下核心能力：

- 模式分离：
  - 相似经历要能被存成彼此区分的独立记忆
- 模式补全：
  - 给局部线索即可恢复整段场景
- 重固化：
  - 每次回忆后允许补充、修订与再组织
- 时间编码：
  - 支持事件链、会话顺序和转折点重建
- 空间编码：
  - 支持地点、布局、环境氛围的场景化编码
- 情绪调幅：
  - 高情绪、高关系价值事件优先固化

## 当前已确认创新点

- 模式分离：
  - 每条记忆生成一个 9D 组合指纹
  - 即使主题相近，只要人物、时间、情感、事件链不同，也要分离为不同 episode
- 模式补全：
  - 用局部线索触发多维召回
  - 补全缺失维度后输出完整场景
- 重固化更新：
  - 保留原始版本
  - 创建新增细节的新版本
  - 用版本链保留记忆演化
- 情绪调幅权重：
  - 高情绪强度记忆获得更深固化
  - 情绪拐点被标记为关键帧
  - 情绪匹配度参与召回排序
- 时序链与事件流：
  - 记忆单元保留前后时刻关系
  - 召回时优先输出时间线叙事

## 关键设计判断

- “维度”应被设计为采集路径与召回路径，而不是仅仅九个字段
- 存储对象应同时支持：
  - 原始输入
  - 发言单元
  - 维度标签
  - 记忆块
  - 图谱关系
- 输出目标不是摘要，而是“场景化还原”
- 首版适合做网页版半自动 demo：
  - 自动提取
  - 人工修订
  - 图谱存储
  - 场景召回

## 当前正式九维

1. 场景维
2. 人物维
3. 时间维
4. 主题维
5. 情感维
6. 关系维
7. 事件维
8. 价值维
9. 来源维

## 当前已确认命名

- `语义维` 不单独占一个九维名额，而是作为“语义理解层”的总称
- `时序维` 正式改为 `时间维`
- 时间维既包含绝对/相对时间，也包含时序逻辑与会话时间锚点
- `证据维` 对外正式名称改为 `来源维`
- `关系维` 已进入九维正式名单
- `图谱层` 不进入九维，单独作为全局关联与重建层

## 当前新增候选

- `状态维`：
  - 用于描述人物、关系、环境、对象在某时间片的状态快照
  - 与 `事件维` 的边界为“发生了什么” vs “当时处于什么状态”
- 生命周期状态枚举：
  - `Active`
  - `Deprecated`
  - `Archived`
- 记忆保留分层候选：
  - `short`
  - `mid`
  - `long`
  - `permanent`

## 当前讨论重点

- 将“类海马体机制”映射到数据流和检索流
- 先明确顶层认知立场与伦理边界
- 区分客观字段与推断字段
- 设计记忆块合成规则
- 设计模式分离与模式补全的索引结构
- 设计重固化更新机制
- 设计召回时的场景重建格式
- 先以网页版 demo 验证交互与数据结构

## 当前新增原则候选

- 系统优先保存“用户对事件与情感的叙事”，而不是宣称保存“真实情感本身”
- 系统角色优先定义为“认知协作者”，而不是“情感诊断者”
- 高风险推断必须附带不确定性与依据链
- 重固化只允许新增版本，不直接覆盖原始记忆
- 情感层、事实层、信念层应分层治理
- 遗忘、降权、封存应与固化同等重要
- “儿童认知”仅作为能力成熟度比喻，不作为产品目标
- 正式建模目标仍是成人社会认知：
  - 成人关系
  - 成人场景
  - 成人社会行为
  - 成人叙事张力
- 引入“字典式记忆块”：
  - 对电影、人物、作品、术语、常识概念等稳定对象生成可复用简介块
  - 后续召回优先直引该稳定块，而不是重新生成解释
- 引入“双脑补架构”：
  - 在线脑补：
    - 对话中一旦提及公共对象，立即激活背景卡
    - 参与当前编码与理解
  - 离线脑补：
    - 会话后回看聊天记录
    - 补充遗漏的字典式记忆块
    - 重算关联关系
    - 为后续召回准备更完整的背景网络
